# ブロック設定例・使用例（examples）ガイドライン

ブロックの全設定パターンをエディターに貼り付けて表示確認できる HTML ファイル群の作成ガイドライン。

## 目的

- マニュアルサイトの「設定・表示例」ページの元データ
- ブロックの全設定を手動で目視確認できる網羅的サンプル
- **block-library 配下のすべてのブロックに適用される恒久ルール**（今後追加されるブロックも含む）

自動テスト用のフィクスチャは別管理（`test/integration/fixtures/blocks/`）。本ディレクトリは「人間がエディターに貼り付けて確認するためのサンプル」と「公開ドキュメント用の元データ」を兼ねる。

## 配置

`src/blocks/block-library/[ブロック名]/examples/[用途名].html`

例:
- `slider/examples/v1-deprecated.html` — v1 → v2 deprecated 変換の手動確認用
- `posts/examples/all-variations.html` — 全設定パターン

## 見出し階層

マニュアルサイトの構造（h1: ページタイトル、h2/h3: 目次出力）を前提とする。

| レベル | 用途 | 例 |
|---|---|---|
| h2 | ページ大見出し | 「設定の組み合わせ例」「設定一覧」 |
| h3 | 設定パネル単位 | 「基本設定」「デザイン設定」 |
| h4 | 設定項目 | 「表示件数の設定例」 |
| h5 | 設定パターン（バリエーション） | 「3件（デフォルト）」 |
| h6 | 原則使用禁止。やむを得ない場合のみ許容 | — |

## ページ構成順

```
冒頭の導入文（<p>）
  ↓
h2「設定の組み合わせ例」
  実用パターン 3〜5 個
  ↓
<!-- wp:separator -->
  ↓
h2「設定一覧」
  h3 → h4 → h5 の構造で全設定を網羅
```

### 「設定の組み合わせ例」セクション

- 用途がイメージできる名前をつける（例: 「ブログのサイドバー向け」「トップページの目玉記事枠」）
- 冒頭に「こんなときに便利です」程度の一言を添える
- 具体設定値を書き切る

### 「設定一覧」セクション

- **h3（パネル）の直後**: そのパネルで何ができるのかを説明する導入文（`<p>`）を配置
- **h4（設定項目）の直後**: その設定の意味・用途を説明する文を配置
- **h5（バリエーション）**: 具体的な設定値がわかる見出し（例: 「3件（デフォルト）」「16:9（デフォルト・横長ワイド）」）
- **セパレーター**: パネル間に`<!-- wp:separator -->`を挿入
- **サイトデータ依存の設定**: 「サイトに合わせて変更してください」等の注記を入れる

## 文章トーン

「仲のいい先輩・ご近所さん」に話すような口調。基本は「です」「ます」調。ビジネス文体は避ける。

| 良い例 | 悪い例（避ける） |
|---|---|
| 「こうすると一覧がぐっと見やすくなりますよ」 | 「視認性が向上します」 |
| 「スマホで見るときは件数を減らしておくと読みやすいです」 | 「モバイルデバイスでは表示件数を削減することが推奨されます」 |
| 「記事の並び順をお好みで変えられます」 | 「記事の並び順を任意に指定できます」 |

- **導入文（h3 直後）**: そのパネルでできることを、用途イメージが湧く形で紹介する
- **設定項目の説明（h4 直後）**: その設定の役割と、どんなときに使うと便利かを軽く添える
- **バリエーション（h5）直後**: 必要に応じて補足する。全バリエーションに文は不要で、違いがわかりにくいときだけ書く

## 網羅基準

- `block.json` の `attributes` に定義された全属性を対象にする
- 各属性のデフォルト値、主要な選択肢、境界値（最小・最大）を含める
- boolean 属性は true/false 両方を含める
- レスポンシブ属性（モバイル/タブレット別設定）がある場合、別設定ありのパターンも含める
- 属性の組み合わせで見え方が変わるもの（例: デザインタイプ × メタ情報表示）はクロスパターンを含める

### 同時に設定できない属性の扱い

- 実装前に `block.json` と edit.tsx の条件分岐（`disabled` / 条件表示）を確認し、排他関係を整理する
- 排他関係がある場合は h5 で「○○モード時の表示」等に分けて網羅する

## ブロック形式

- **ダイナミックブロック**: `<!-- wp:ystdtb/[ブロック名] {"attr":"value"} /-->`
- **通常ブロック**: `<!-- wp:ystdtb/[ブロック名] {"attr":"value"} -->` + innerHTML + `<!-- /wp:ystdtb/[ブロック名] -->`
- **インナーブロックを持つブロック**: 親ブロック内に子ブロックを含めた完全な HTML を記述。サンプルテキストには日本語のダミーテキストを使用

### 親子ブロックの扱い

親ブロック側に 1 ファイルで集約する（マニュアルサイトが親ブロック単位でページを組む前提）。子ブロックの設定も親の HTML 内で網羅する。

## 作業フロー（1 ブロックあたり）

1. `block.json` の attributes を全て書き出す
2. edit.tsx / inspector-controls を読み、パネル構成と排他関係を整理する
3. パネル → 設定項目 → バリエーションの**見出しツリー**を設計書に落とす（実装前に確認を取る）
4. HTML 執筆
5. 実機（wp-env）のエディターに貼り付け、表示確認
6. フロント表示確認
