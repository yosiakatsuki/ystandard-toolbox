# aktk-block-components コンポーネント作成ガイドライン

aktk-block-components は yStandard シリーズ（yStandard Toolbox、yStandard Blocks）で共用するコンポーネントライブラリである。
コンポーネント作成時はプラグイン固有のロジックを含めないよう注意すること。

---

## テキストドメイン

**テキストドメインは `aktk-block-components` を使用する。**

```tsx
// ✅ 正しい
__( '画像サイズ', 'aktk-block-components' )

// ❌ 誤り（プラグイン固有のテキストドメイン）
__( '画像サイズ', 'ystandard-toolbox' )
```

aktk-block-components は複数プラグインで共有されるため、特定プラグインのテキストドメインを指定してはならない。

以下のファイルでは歴史的経緯により `ystandard-toolbox` が使用されている。新規作成時は `aktk-block-components` を使用すること。

## CSSクラスプレフィックス

**`aktk-component__` を使用する（単数形）。**

```css
/* ✅ 正しい */
.aktk-component__image-size-select { }

/* ❌ 誤り（プラグイン固有のプレフィックス） */
.ystdtb-image-size-select { }
.ystd-component__image-size-select { }
```

### 既存コンポーネントの状況

`aktk-components__`（複数形）を使用しているコンポーネントもあるが、新規作成時は `aktk-component__`（単数形）に統一する。

| パターン | 例 | 備考 |
|---|---|---|
| `aktk-component__*` | `aktk-component__icon-select` | 推奨（新規はこちら） |
| `aktk-components__*` | `aktk-components__horizon-buttons` | 旧パターン（既存は維持） |

## CSSファイル

**コンポーネントにつき `index.css` 1ファイルに統一する。**

- CSSが必要な場合、コンポーネントディレクトリ直下に `index.css` を作成
- TSXファイルからCSS importは**行わない**（共有CSSとして一括配信される）
- `src/aktk-block-components/index.css` に `@import` を追加する
- CSS Nesting 構文を使用可能（postcss-preset-env がフラットCSSに展開する）

```css
/* コンポーネント名/index.css */
.aktk-component__example {
	& .child {
		color: red;
	}
}
```

詳細は `コンポーネントCSS移行.md` を参照。

## ディレクトリ構成

```
src/aktk-block-components/components/[コンポーネント名]/
├── index.tsx       # エントリーポイント（export）
├── index.css       # CSS（必要な場合のみ）
├── types.ts        # 型定義（必要な場合のみ）
└── [その他].tsx    # 内部コンポーネント・ユーティリティ
```

## importパス

aktk-block-components 内の相互参照はエイリアスを使用する。

```tsx
// ✅ エイリアス（推奨）
import { CustomSelectControl } from '@aktk/block-components/components/custom-select-control';
import BaseControl from '@aktk/block-components/wp-controls/base-control';

// ❌ 相対パス（非推奨）
import { CustomSelectControl } from '../../custom-select-control';
```

---

## @wordpress/components の直接利用不可

`@wordpress/components` のコンポーネントは直接使用しない。`wp-controls/` 内にラップしたコンポーネントを使用する。

```tsx
// ✅ 正しい
import BaseControl from '@aktk/block-components/wp-controls/base-control';
import ToggleControl from '@aktk/block-components/wp-controls/toggle-control';

// ❌ 誤り
import { BaseControl, ToggleControl } from '@wordpress/components';
```

**例外**: `wp-controls/` にラップが存在しない場合は、必要に応じて `wp-controls/` にラップを新規作成する。

---

## BaseControl によるラップ規則

`BaseControl` によるラップは**コンポーネントの呼び出し側（ブロック等）で行う**。コンポーネント内部では `BaseControl` をラップしない。

```tsx
// ブロック側（呼び出し側）での使用例
<BaseControl>
	<ImageSizeSelect value={ value } onChange={ onChange } />
</BaseControl>
```

---

## 新規コンポーネント追加の判断基準

- aktk-block-components は複数プロジェクトで共有されるため、**プラグイン固有のロジックを含めない**
- 汎用的に利用できるコンポーネントのみ追加する
- ブロック固有のロジック（そのブロックでしか使わない処理）は `src/blocks/` 側に作成する
- 判断に迷う場合は相談すること
